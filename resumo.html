Relatório de Testes com GenieACS

Resumo Geral

Três dispositivos foram testados no ambiente GenieACS:

  • ZTE ONT
  • Greatek V1
  • Greatek V2

Interface Web (:3000)

Todos os dispositivos são exibidos como online, mas apresentam falhas ao interagir pela interface web:

  • Greatek: ao tentar realizar alterações, o sistema exibe Device is offline.
  • ZTE ONT: apresenta erros de parâmetros ao tentar alterar configurações.

Via API (:7557)

Mesmo com erros similares, após reiniciar os dispositivos, as tarefas são aplicadas corretamente.

ZTE ONT

  • Muito instável com ajustes de configurações.
  • Pouco flexível.
  • Diferenças principais: wlan1 (2.4GHz) e wlan5 (5.8GHz).

Greatek V2

Apresenta instabilidade semelhante ao ZTE. Exemplo de comportamento durante testes:

  • API chamada para renomear SSID e alterar senha.
  • API chamada para reinício do dispositivo (sem efeito imediato).
  • API usada para mudar o canal e novo reboot (sem efeito).
  • API usada para forçar captura de parâmetros (sem efeito).
  • Tasks permaneciam na fila.
  • Ao reiniciar manualmente, todas as tasks foram executadas em sequência.

Conclusão: execução da fila é extremamente lenta ou dependente de reboot manual.

Greatek V1

  • Melhor desempenho entre os testados.
  • Alterar SSID e senha via API seguido de reboot funciona conforme esperado.
  • Interface Web também funciona bem com os parâmetros sendo atualizados automaticamente após reboot.
